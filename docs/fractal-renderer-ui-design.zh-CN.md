本文档记录候选 C 作为 FractalForge 通用分形渲染器界面的产品设计方向。

## 产品形态

FractalForge 应该像一个紧凑、专业的 macOS 创作工具，而不是宣传页。
打开后的第一屏就是可工作的渲染界面：

```text
+--------------------------------------------------------------------------------+
| 工具栏：算法 | 重置 | 保存预设 | 导出 | 性能开关                               |
+--------------------+-----------------------------------------+-----------------+
| 算法库             |                                         | 参数面板        |
| - Mandelbrot       |                                         | 视口            |
| - Julia            |          Metal 分形渲染画布              | 迭代            |
| - Burning Ship     |                                         | 公式参数        |
| - Newton           |                                         | 颜色            |
| - Multibrot        |                                         | 渲染            |
| - Mandelbox        |                                         |                 |
+--------------------+-----------------------------------------+-----------------+
| FPS | 迭代次数 | 精度模式 | GPU 内存 | 渲染耗时                                |
+--------------------------------------------------------------------------------+
```

## 主要区域

- 中央画布：Metal 驱动的大型分形视口，支持滚轮缩放、拖拽平移、双击重置、实时预览和深度缩放。
- 左侧边栏：可选择的分形算法库。
- 右侧检查器：根据当前算法变化的参数面板。
- 顶部工具栏：全局命令和快速算法切换。
- 底部状态栏：实时渲染指标和性能状态。

## 算法库

- Mandelbrot：默认复平面探索器。
- Julia：由复数常量 `c` 驱动的参数化分形族。
- Burning Ship：使用绝对值变换的 Mandelbrot 变体。
- Newton：根求解吸引域可视化。
- Multibrot：通过指数/幂次推广的 Mandelbrot。
- Mandelbox：后续可扩展的 3D 折叠分形候选。

## 参数面板

- 视口：中心 X/Y、缩放比例、旋转、缩放速度。
- 迭代：最大迭代次数、逃逸半径、精度模式。
- 公式参数：Julia 实部/虚部常量、Multibrot 幂次、Newton 根模式。
- 颜色：调色板、对比度、平滑、曝光、背景色。
- 渲染：分辨率比例、抗锯齿、FPS 上限、实时预览。

## 组件状态

- 算法行：普通、悬停、选中。
- 画布：空闲、渲染中、深度缩放、暂停。
- 参数控件：普通、聚焦、禁用、无效输入。
- 窗口：可见、隐藏、最小化。
- 工具栏命令：普通、悬停、按下、禁用。

## 视觉方向

- 背景：近黑色应用框架、深灰面板、高对比度画布。
- 强调色：使用系统蓝表示选中控件和激活命令。
- 字体：SF Pro 风格，标签紧凑，指标数字使用等宽字体。
- 控件：分段选择器、滑块、步进器、数值输入框、开关、菜单。
- 图标：使用 SF Symbols 或 lucide 风格图标表示重置、导出、性能、预设。

## 实现映射

- `FractalKind` 和 `FractalDefinition` 驱动边栏算法列表和检查器显示逻辑。
- `FractalViewport` 持有当前算法、中心点、缩放比例和重置逻辑。
- `MetalRenderer` 通过 uniforms 接收当前所有分形参数。
- `ContentView` 应从当前的右上角叠加控件，演进成稳定的三栏桌面工具布局。
- 未来新增算法参数时，应先加入 definition 层，再暴露到右侧检查器。
